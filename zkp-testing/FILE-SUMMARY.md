#### `run-all-tests.js` â­ **MAIN FILE**

**Purpose:** Master test runner yang menjalankan semua testing suite

**Features:**

---

#### `test-functional.js`

**Purpose:** Pengujian fungsionalitas sistem (Tabel 4.2 Bab 4)

**Test Cases:**

- TC-01: Registrasi normal
- TC-02: Login valid
- TC-03: Password salah
- TC-04: TOTP salah
- TC-05: Password benar, TOTP salah
- TC-06: Username duplikat
- TC-07: User tidak terdaftar
- TC-08: TOTP kadaluarsa

**Metrics:**

- Total tests run
- Pass/Fail count
- Success rate (%)

**Output:** `functional-test-results.json`

---

#### `test-replay-attack.js`

**Purpose:** Pengujian keamanan terhadap replay attack (Tabel 4.3 & 4.5)

**Test Scenarios:**

- SEC-01: Replay ZKP immediate
- SEC-02: Replay TOTP in time window
- SEC-03: Full packet replay
- SEC-04: SessionID replay
- SEC-05: Expired TOTP replay
- SEC-06: Password leakage test
- SEC-07: Database breach simulation

**Metrics:**

- Secure vs Vulnerable count
- Security score (%)
- Risk assessment (Protected/Medium/High/Critical)

**Output:** `replay-attack-test-results.json`

---

#### `test-performance.js`

**Purpose:** Pengujian performa sistem (Tabel 4.7 & 4.9)

**Performance Tests:**

- PERF-01: Client-side key generation
- PERF-02: Registration time
- PERF-03: Challenge request
- PERF-04: ZKP proof generation
- PERF-05: Complete login time
- PERF-06: Concurrent load (100 users)
- PERF-07: Server-side verification
- PERF-08: Overhead comparison

**Metrics:**

- Mean, std dev, min, max
- P50, P95, P99 percentiles
- Success rate under load
- Client vs server overhead distribution

**Output:** `performance-test-results.json`
